Markdown is the standard format for technical writing. Here is how to use it effectively.

## Structure First

Start with headings. A good document has a clear hierarchy:

- `#` Title (one per document)
- `##` Main sections
- `###` Subsections

## Code Blocks

Always specify the language for syntax highlighting:

```python
def greet(name: str) -> str:
    return f"Hello, {name}!"
```

## Tables

Use tables for comparisons:

| Tool | Use Case |
|------|----------|
| Markdown | Documentation |
| YAML | Configuration |
| JSON | Data exchange |

## Links and Images

Use descriptive link text:

```markdown
Read the [official documentation](https://example.com)
```

Clear documentation is as important as clean code.
